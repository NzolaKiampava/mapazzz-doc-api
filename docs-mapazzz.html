Introdução

Bem-vindo à documentação da API do Sistema de Detecção de Zonas de Risco de Malária. Esta API fornece acesso a dados geoespaciais, análises preditivas e informações em tempo real sobre áreas com risco de transmissão de malária.

Com esta API, você pode:

  • Acessar mapas de calor de risco de malária por região
  • Obter dados históricos e atuais de casos notificados
  • Consultar dados climáticos correlacionados com a incidência de malária
  • Receber previsões de surtos com base em modelos de aprendizado de máquina
  • Configurar e gerenciar alertas para áreas específicas
Nota: Esta API é atualizada diariamente com dados de fontes oficiais de saúde e meteorologia.

Autenticação

Todas as requisições à API requerem autenticação usando uma chave de API (API Key). Para obter sua chave, entre em contato com nossa equipe de suporte.

Autenticação via Header

Inclua sua chave de API no cabeçalho HTTP X-API-Key:

X-API-Key: sua_chave_api_aqui

Autenticação via Query Parameter

Alternativamente, você pode incluir sua chave como parâmetro de consulta:

https://api.malariarisk.com/v1/risk-zones?apiKey=sua_chave_api_aqui
Importante: Mantenha sua chave de API segura. Não compartilhe ou exponha sua chave em repositórios públicos ou código-fonte cliente.

Endpoints

Mapas de Risco

GET /v1/risk-zones

Retorna um mapa de zonas de risco de malária com base em coordenadas geográficas.

Parâmetros de Consulta

Parâmetro Tipo Obrigatório Descrição
latitude Number Sim Latitude do ponto central da busca
longitude Number Sim Longitude do ponto central da busca
radius Number Não Raio de busca em quilômetros (padrão: 10)
resolution String Não Resolução do mapa: "high", "medium", "low" (padrão: "medium")

Exemplo de Resposta

// Status: 200 OK
{
  "status": "success",
  "data": {
    "center": {
      "latitude": -3.1190,
      "longitude": -60.0217
    },
    "radius": 10,
    "resolution": "medium",
    "riskZones": [
      {
        "id": "zone-123",
        "riskLevel": "high",
        "coordinates": [
          {
            "latitude": -3.1150,
            "longitude": -60.0250
          },
          // ... mais coordenadas formando o polígono
        ],
        "caseCount": 126,
        "lastUpdated": "2025-04-15T10:30:00Z"
      },
      // ... mais zonas de risco
    ]
  }
}

POST /v1/cases/report

Reporta um novo caso de malária ao sistema.

Corpo da Requisição (JSON)

{
"location": {
"latitude": -3.1150,
"longitude": -60.0250,
"regionId": "region-456"
},
"reportDate": "2025-04-17T14:30:00Z",
"parasiteType": "P. vivax",
"patientData": {
"ageGroup": "31-50",
"gender": "M"
},
"status": "confirmed"
}

Parâmetros do Corpo

Parâmetro Tipo Obrigatório Descrição
location.latitude Number Sim Latitude do local do caso
location.longitude Number Sim Longitude do local do caso
location.regionId String Não ID da região, se conhecido
reportDate String (ISO) Sim Data do reporte (formato: YYYY-MM-DDThh:mm:ssZ)
parasiteType String Sim Tipo de parasita: "P. vivax", "P. falciparum", "Mixed", "Unknown"
patientData.ageGroup String Sim Faixa etária: "0-5", "6-17", "18-30", "31-50", "51+"
patientData.gender String Não Gênero: "M", "F", "Other"
status String Sim Status do caso: "confirmed", "suspected"

Exemplo de Resposta

// Status: 201 Created
{
"status": "success",
"data": {
"id": "case-790",
"message": "Caso reportado com sucesso",
"reportedAt": "2025-04-17T14:32:00Z"
}
}

Sistema de Alertas

POST /v1/alerts/subscribe

Inscreve um usuário para receber alertas de risco de malária em uma área específica.

Corpo da Requisição (JSON)

{
"email": "user@example.com",
"location": {
"latitude": -3.1190,
"longitude": -60.0217,
"radius": 10
},
"riskThreshold": "high",
"frequency": "daily"
}

Parâmetros do Corpo

Parâmetro Tipo Obrigatório Descrição
email String Sim Endereço de e-mail para receber alertas
location.latitude Number Sim Latitude do centro da área monitorada
location.longitude Number Sim Longitude do centro da área monitorada
location.radius Number Sim Raio da área em quilômetros
riskThreshold String Sim Nível de risco para disparar alerta: "low", "medium", "high"
frequency String Sim Frequência dos alertas: "daily", "weekly"

Exemplo de Resposta

// Status: 201 Created
{
"status": "success",
"data": {
"subscriptionId": "sub-123",
"message": "Inscrição realizada com sucesso",
"createdAt": "2025-04-18T10:00:00Z"
}
}

Códigos de Resposta

A API utiliza códigos de status HTTP padrão para indicar o sucesso ou falha das requisições.

Código Descrição
200 OK Requisição bem-sucedida
201 Created Recurso criado com sucesso (usado em POST)
400 Bad Request Parâmetros inválidos ou ausentes
401 Unauthorized Chave de API inválida ou ausente
403 Forbidden Acesso negado ou limite de requisições excedido
404 Not Found Recurso não encontrado
500 Internal Server Error Erro interno do servidor

Exemplo de Resposta de Erro

// Status: 400 Bad Request
{
"status": "error",
"error": {
"code": 400,
"message": "Parâmetro 'latitude' é obrigatório"
}
}

Exemplos

Exemplo em Python: Consultar Zonas de Risco

import requests

API_KEY = "sua_chave_api_aqui"
BASE_URL = "https://api.malariarisk.com/v1"

headers = {
"X-API-Key": API_KEY
}

params = {
"latitude": -3.1190,
"longitude": -60.0217,
"radius": 10,
"resolution": "medium"
}

response = requests.get(f"{BASE_URL}/risk-zones", headers=headers, params=params)

if response.status_code == 200:
data = response.json()
print(data)
else:
print(f"Erro: {response.status_code} - {response.json()['error']['message']}")

Exemplo em JavaScript: Reportar um Caso

const API_KEY = "sua_chave_api_aqui";
const BASE_URL = "https://api.malariarisk.com/v1";

const headers = {
"X-API-Key": API_KEY,
"Content-Type": "application/json"
};

const body = {
location: {
latitude: -3.1150,
longitude: -60.0250,
regionId: "region-456"
},
reportDate: "2025-04-17T14:30:00Z",
parasiteType: "P. vivax",
patientData: {
ageGroup: "31-50",
gender: "M"
},
status: "confirmed"
};

fetch(`${BASE_URL}/cases/report`, {
method: "POST",
headers: headers,
body: JSON.stringify(body)
})
.then(response => response.json())
.then(data => console.log(data))
.catch(error => console.error("Erro:", error));
